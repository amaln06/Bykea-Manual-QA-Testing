# Bykea-Manual-QA-Testing
This project is part of Software Quality Assurance (SQA) practice, focusing on manual testing of the Bykea Android application. The objective is to validate system functionality, verify compliance with requirements, and identify defects in UI, navigation, and functional modules.
Testing was performed using black-box testing techniques without access to source code.

## Objectives
1. Validate system functionality against requirements
2. Perform Functional Testing of core modules
3. Ensure proper System Behavior under normal and abnormal conditions
3. Identify UI/UX issues including cosmetic defects
4. Evaluate application performance under basic conditions
5. Ensure smooth user experience (Usability Testing)
6. Detect and report defects with proper classification

## Modules Under Test
1. Authentication / Login Module
2. Dashboard / Home Screen
3. Location Selection Module (Pickup/Drop-off)
4. Menu Bar Module

## Test Design Approach
Test cases were designed using: 
1. Requirement-based testing
2. Exploratory testing
3. Boundary and negative scenarios
4. Real-world user behavior simulation

## Defect Classification
During testing, defects are categorized based on:
### Severity Levels:
High (System crash / major functionality failure)
Medium (Functional issue affecting workflow)
Low (Minor issue / cosmetic issue)
### Priority Levels:
High Priority (Needs immediate fix)
Medium Priority
Low Priority

## Conclusion 
This project is ongoing and will continue towards End-to-End Testing (E2E Testing), including deeper performance evaluation and usability enhancement analysis.
